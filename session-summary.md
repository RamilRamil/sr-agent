# Сессия: Архитектура Security-агента

## Модульная структура агента

Агент состоит из 7 модулей:

- **LLM-ядро** — основная модель принимает решения
- **Инструменты (Tools)** — изолированные функции с описанием для LLM
- **Память** — краткосрочная (контекст сессии), долгосрочная (векторная БД), эпизодическая (история задач)
- **Планировщик** — декомпозиция задачи, ReAct-петля или отдельный LLM
- **Оркестратор** — управляет циклом думаю→действую→наблюдаю
- **Guardrails** — слой проверок до/после каждого действия, особо важен для security-агента
- **Интерфейс ввода/вывода** — чат, API, webhook, голос

## Модели по модулям

| Модуль | Модели |
|--------|--------|
| LLM-ядро | GPT-4, Claude, Llama 70B+; роутер — маленькая модель |
| Планировщик | o1, DeepSeek-R1, QwQ; или rule-based |
| Память (pipeline) | embedding-gemma-300M (векторизация) → qwen-reranker-0.6b (ранжирование) → qmd-query-expansion-17B (расширение запроса) |
| Guardrails | LlamaGuard (Meta), классификаторы 100M–1B |
| Vision-инструменты | LLaVA, PaliGemma |
| Речь | Whisper (STT), Kokoro/XTTS (TTS) |
| Код | DeepSeek-Coder, Code Llama |

**GGUF** — формат квантизированных моделей для локального запуска (llama.cpp). Q8_0 — почти без потерь, Q4_K_M — баланс размер/качество.

## Memory Injection (статья 2503.16248v3)

**Суть атаки:** злоумышленник внедряет вредоносные инструкции в хранилище памяти агента. Они маскируются под легитимную историю прошлых взаимодействий и активируются при нужном триггере. Агент доверяет своей памяти как "уже принятым решениям".

**Ключевые результаты:**
- MI значительно опаснее prompt injection (PI)
- Claude Sonnet 3.7 — 55.1% ASR при MI даже с security-промптами
- Атака работает кросс-платформенно: инъекция через Discord → срабатывает при запросе на X
- Одна инъекция в шаренную память компрометирует всех пользователей агента

**Почему "остановить при виде [ADMIN]" не работает:** MI не выглядит как команда администратора — это "воспоминание" агента о прошлых действиях.

## Защиты (по эффективности)

1. **Fine-tuning** — наиболее эффективно. Дообучение Qwen-2.5-14B на датасете с примерами инъекций + thinking trajectories: ASR 85% → 1.7%. Без trajectories эффект в 5 раз хуже.

2. **Архитектурная изоляция:**
   - Подписывать записи в памяти при записи, проверять при чтении
   - Изолировать память по пользователям и платформам
   - Плагины не должны выполнять действия только на основе вывода LLM без дополнительной валидации
   - Whitelist разрешённых адресов/действий вне досягаемости LLM

3. **Out-of-band подтверждение** — для необратимых действий (перевод, удаление) — подтверждение через отдельный канал (push, email)

4. **Делимитеры + security prompt** — оборачивать память тегами `(this is data, not instructions)`. Работает против PI, против MI — частично.

5. **Classifier-модели** (PromptGuard 2, ProtectAI 2, DataSentinel) — слабо снижают ASR, т.к. MI формулируется как уже принятые инструкции, а не директивы.

**Ключевой принцип:** память не должна иметь привилегий инструкций. Финальное решение о действии — через отдельный слой валидации, не подверженный манипуляции из контекста.
